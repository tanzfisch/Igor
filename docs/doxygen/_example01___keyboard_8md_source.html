1 Keyboard Usage Example {#example01}
2 ======================
3 
4 In this tutorial we have a look at Keyboard input and how to set up a window.

13 KeyboardExample class
14 =====================
15 
16 Let's have a brief look at the header. So there is two new includes. iWindow because we gonna need a window to receive the keyboard messages and iKeyCodeMap because the callbacks to handle the keyboard messages will receive a type called iKeyCode which is Igor specific key codes.
17 
18  #include <iWindow.h>
19  #include <iKeyCodeMap.h>
20  using namespace Igor;
21 
22 Next we have a member variable of type iWindow which is the window's instance that we gonna use.
23 
24  iWindow _window;
25 
26 And there are a couple of event handlers related to keyboard and window events.
27 
28  void onKeyPressed(iKeyCode key);
29  void onKeyReleased(iKeyCode key);
30  void onKeyASCIIInput(char c);
31  void onKeyESCPressed();
32  void onCloseWindow();
33 
34 Let's have a look at the implementation part now. So there is an other new include this time for the actual Keyboard.
35 
36  #include <iKeyboard.h>
37  using namespace Igor;
38 
39 We need that so we can register our callbacks to some key press or release events. Which happens in our init method. Let's have a look at that init method. First we setup and open a window.
40 
41  // init window
42  _window.setTitle("Keyboard Example");
43  // opening the window will result in a warning since we have no view to render to defined. but for this example we don't need anything to render
44  _window.open();
45  // need to know when the window was closed so we can shut down the application
46  _window.registerWindowCloseDelegate(WindowCloseDelegate(this, &KeyboardExample::onCloseWindow));
47 
48 As you probably noticed there is a warning during runtime. I means that the window was opened even though there was no view attached to it. So it is allowed to have a window without a view but in generall it does not make sence there for this warning reminds you of that when you accidentally forgot to register a view to the window.
49 
50 And than we register some callbacks to keyboard events.
51 
52  // registers a callback to the key pressed event. called when any key was pressed
53  iKeyboard::getInstance().registerKeyDownDelegate(iKeyDownDelegate(this, &KeyboardExample::onKeyPressed));
54  // registers a callback to the key release event. called when any key was released
55  iKeyboard::getInstance().registerKeyUpDelegate(iKeyUpDelegate(this, &KeyboardExample::onKeyReleased));
56  // registers a callback to the specific key pressed event. In this case the ESC key. called only when the esc key was pressed
57  iKeyboard::getInstance().registerKeyDownDelegate(iKeyDownSpecificDelegate(this, &KeyboardExample::onKeyESCPressed), iKeyCode::ESC);
58  // registers a callback to the ASCII input event. called for every ascii input event
59  iKeyboard::getInstance().registerKeyASCIIDelegate(iKeyASCIIDelegate(this, &KeyboardExample::onKeyASCIIInput));
60 
61 So that's it for the initialisation. Let's have a look at the deinit method too.
62 
63  // unregisters all callbacks. in principle this is optional because we shut down here anyway but if you want a clean shutdown you should do that
64  iKeyboard::getInstance().unregisterKeyASCIIDelegate(iKeyASCIIDelegate(this, &KeyboardExample::onKeyASCIIInput));
65  iKeyboard::getInstance().unregisterKeyDownDelegate(iKeyDownDelegate(this, &KeyboardExample::onKeyPressed));
66  iKeyboard::getInstance().unregisterKeyUpDelegate(iKeyUpDelegate(this, &KeyboardExample::onKeyReleased));
67  iKeyboard::getInstance().unregisterKeyDownDelegate(iKeyDownSpecificDelegate(this, &KeyboardExample::onKeyESCPressed), iKeyCode::ESC);
68 
69  // closes the window if it was not closed already
70  _window.close();
71  // unregisters an other callback
72  _window.unregisterWindowCloseDelegate(WindowCloseDelegate(this, &KeyboardExample::onCloseWindow));
73 
74 Pretty self explanatory. Also in the implementation of our callbacks there is nothing suprising except for onCloseWindow.
75 
76  void KeyboardExample::onCloseWindow()
77  {
78  con_endl("windows was closed");
79  // stop the application of the window was closed.
80  // because once the window is closed we loose the keyboard input and we have to close the console manually (with e.g. Ctrl+C)
81  iApplication::getInstance().stop();
82  }
83 
84 So basically when the window was closed we have to shutdown the application too. Because otherwise the application would continue without having a window and therefore we don't get any more calls to our callbacks.
85 
86 In an other scenario when we don't need a window that would be fine or course.
87 
88 Besides using events the Keyboard device's key states can be retrived any time directly from the iKeyboard class. See the Interface documentation for that.
89 And that's it. This is how we setup a window and some keyboard event handlers/callbacks/delegates. For further questions please contact [tanzfisch](https://github.com/tanzfisch).
