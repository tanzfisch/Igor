1 First Steps Example {#example00}
2 ===================
3 
4 In this very first example is explained how to get Igor up and running for the first time. Where to get it and what is minimum needed to start loop a bit and end an application using Igor.
5 
6 Build
7 =====
8 
9 The code for this example can be found in the folder examples/00 FirstSteps. In a vs[ver] subfolder you will find a visual studio solution which contains example project but also all dependent projects. So you just open this one and than build and start. If everything went right you gonna see the following screen. If not please don't hasitate to contact [tanzfisch](https://github.com/tanzfisch).

13 Main
14 ----
15 
16 So what is it doing. Let's open main.cpp first.
17 
18  #include "FirstSteps.h"
19 
20  #include <Igor.h>
21  using namespace Igor;
22 
23  int main(void)
24  {
25  /*! call this before you call anything else of Igor
26  */
27  Igor::startup();
28 
29  // create and run first steps class
30  FirstSteps* firstSteps = new FirstSteps();
31  firstSteps->run();
32  delete firstSteps;
33 
34  /*! call this after you are done with using Igor
35  */
36  Igor::shutdown();
37 
38  return 0;
39  }
40 
41 Besides including Igor.h you might notice only two Igor related commands are called. Igor::startup and Igor::shutdown.
42 It is important that all other calls to Igor are made between those calls. Or in other words the very first thing to call from Igor is startup and the very last to call is shutdown.
43 
44 Igor's startup is not doing much more then initializing the console and reading a configuration file if available but shutdown is doing a lot of cleanup and checking to give you a report e.g. about not release resources.
45 
46 FirstSteps class
47 ----------------
48 
49 The class called FirstSteps is what we are going to have a look next. It is just personal flavor to put everything in that class you can also directly start making calls to Igor.
50 
51 So there is really nothing interesting inside the header file. So we can skip that and have a look at the source file FirstSteps.cpp.
52 
53  #include <iApplication.h>
54  #include <iTimer.h>
55  using namespace Igor;
56 
57  // let's use the console to see what happens
58  #include <iaConsole.h>
59  using namespace IgorAux;
60 
61 So here we have some more includes. First is iApplication.h. iApplication is handles basically the main loop of Igor once that one is started it will only end if said so.
62 Second is iTimer.h. That one contains timing functionality. It can tell you the current time since system start or the delta time since the last frame.
63 And then there is the iaConsole.h which is our logging console. Here you might notice that it's within a different namespace called IgorAux. That's because the most common functionality of Igor was moved to a separate library so one can use it independently from Igor.
64 
65 Let's have a look at the constructor of the FirstSteps class.
66 
67 `con_endl("ctor");`
68 
69 The very first thing is using the console to print some status on the console. `con_endl` is a _normal_ message that has a line break in the end. Check out iConsole.h for more interfaces like con_assert, con_debug, con_err, con_warn and more.
70 
71 The second line is way more interessting. It's about registering a callback to the main loop of our application.
72 
73  iApplication::getInstance().registerApplicationHandleDelegate(ApplicationHandleDelegate(this, &FirstSteps::onHandle));
74 
75 It creates a delegate with the method `onHandle` of `this` instance and passes it the a register method of our singleton instance of the class iApplication. Internally the delegate will be bound to an event. And when ever this event is triggered the delegate will be called (it's basically something like a signal and slot mechanism).
76 
77 In the destructor of our class you can see the the handle being unregistered again. This way we keep every thing consistent for a clean shutdown.
78 
79 Let's have a look at the next method called run.
80 
81  void FirstSteps::run()
82  {
83  con_endl("run");
84 
85  // starts the applications endless loop
86  iApplication::getInstance().run();
87  }
88 
89 It basically does only one thing. It calls the applications run method which contains the main loop of Igor. `run` will not return until we tell Igor to stop. And we gonna do that in our last method called onHandle.
90 
91 As said before the onHandle method will be called by Igor every frame. So we just use this to simply let it run a few times controlled by a counter and then make Igor stop by calling `iApplication::getInstance().stop();`.
92 
93  void FirstSteps::onHandle()
94  {
95  // since there is nothing to do for the application besides init and deinit some internal stuff
96  // the handle will be called es often as possible. which should be reflected in a verry high frame rate
97 
98  // lets count
99  _counter++;
100 
101  // print the counter
102  con_endl("handle " << _counter);
103 
104  if (_counter >= 5)
105  {
106  // breaks the applications endless loop
107  iApplication::getInstance().stop();
108  con_endl("ms since the programm started " << iTimer::getInstance().getTime() - _time);
109  }
110  }
111 
112 As alternative to using iApplication::run which starts Igors main loop you can implement your own main loop and call iApplication::iterate in it. You can find an example for that in main.cpp.
113 
114 And that's already it. This is how we setup Igor, start and stop it again. For further questions please contact [tanzfisch](https://github.com/tanzfisch).
